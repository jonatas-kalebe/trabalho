### Fase 1: Análise Léxica (O arquivo Flex)
Esta fase transforma o código-fonte em *tokens*.
* [cite_start]**O que pedir para a IA:** "Crie um arquivo `g-v1.l` para o Flex[cite: 207]. [cite_start]Quero expressões regulares para identificar os tokens definidos na linguagem G-V1, como palavras reservadas (PRINCIPAL, INT, CAR), operadores, e os padrões de IDENTIFICADOR e CADEIACARACTERES[cite: 58]."
* [cite_start]**Detalhes críticos para focar:** * A função `yylex()` gerada deve retornar o token e salvar lexemas em `yytext`[cite: 67, 69].
    * [cite_start]É obrigatório ignorar comentários (que começam com `/*` e terminam com `*/`) mesmo que ocupem várias linhas[cite: 70, 71].
    * [cite_start]O tratamento de erros deve ser exato: imprimir `ERRO: ` seguido de `CARACTERE INVÁLIDO`, `COMENTÁRIO NAO TERMINA` ou `CADEIA DE CARACTERES OCUPA MAIS DE UMA LINHA`, além do número da linha[cite: 75, 76].

### Fase 2: Análise Sintática (O arquivo Bison)
Esta fase valida a estrutura do código com base na gramática.
* [cite_start]**O que pedir para a IA:** "Crie um arquivo `g-v1.y` para o Bison[cite: 208]. [cite_start]Transcreva a gramática LALR da linguagem G-V1, mapeando declarações de variáveis, blocos e expressões[cite: 14, 20, 91]."
* **Detalhes críticos para focar:**
    * [cite_start]O Bison precisará das declarações `extern int yylineno;`, `extern char* yytext;` e `extern int yylex();` para se comunicar corretamente com o Flex[cite: 99, 100, 101].
    * [cite_start]A função de erro `yyerror()` deve ser customizada para exibir a mensagem iniciando com a palavra `ERRO: ` e a linha do problema[cite: 94, 95].

### Fase 3: Árvore Sintática Abstrata (AST)
Aqui, o Bison deixará de apenas validar o código e passará a construir uma estrutura de dados em memória.
* [cite_start]**O que pedir para a IA:** "Crie as `structs` ou classes em C/C++ para representar a AST da G-V1[cite: 122]. [cite_start]Depois, adicione as ações semânticas no arquivo do Bison para instanciar e conectar os nós à medida que a gramática é reduzida[cite: 123]."
* **Detalhes críticos para focar:**
    * [cite_start]A árvore precisa ser compacta[cite: 125].
    * [cite_start]É essencial que os nós armazenem a linha onde os itens léxicos ocorreram para futuras mensagens de erro[cite: 126].

### Fase 4: Tabela de Símbolos e Escopo
Esta é a estrutura de dados que gerencia a visibilidade das variáveis.
* [cite_start]**O que pedir para a IA:** "Implemente em C/C++ uma Pilha de Tabelas de Símbolos[cite: 141]. [cite_start]Ela deve ter operações para: iniciar a pilha, empilhar um novo escopo, pesquisar um nome (do topo para a base) e remover o escopo atual[cite: 143, 144, 145, 146, 148]."
* **Detalhes críticos para focar:**
    * [cite_start]Sempre que um bloco inicia com declarações, um novo escopo é criado e empilhado[cite: 138]. [cite_start]Variáveis locais se sobrepõem às globais do mesmo nome durante a duração do bloco[cite: 163].

### Fase 5: Analisador Semântico
Este passo percorre a AST garantindo que o programa faz sentido logicamente.
* [cite_start]**O que pedir para a IA:** "Escreva uma função que percorre a AST do bloco mais externo para o mais interno[cite: 180]. [cite_start]Integre esta função com a Pilha de Tabelas de Símbolos para validar declarações e checar tipos[cite: 182, 186]."
* **Detalhes críticos para focar:**
    * [cite_start]Uma variável só pode ser usada se já foi declarada[cite: 164].
    * [cite_start]Operações entre tipos diferentes (`int` e `car`) são proibidas[cite: 166, 167]. 
    * [cite_start]Se houver falha, reporte o erro e termine a compilação[cite: 190].

### Fase 6: Geração de Código MIPS
[cite_start]A última etapa lógica do compilador, executada apenas se não houver erros semânticos[cite: 194].
* [cite_start]**O que pedir para a IA:** "Crie um segundo percurso na AST para gerar código[cite: 194, 195]. [cite_start]Use a tabela de símbolos para calcular a posição de memória e o tamanho das variáveis [cite: 198] [cite_start]e emita as instruções em Assembly MIPS[cite: 201]."

### Fase 7: Automação (Makefile)
Para o professor conseguir compilar e testar.
* [cite_start]**O que pedir para a IA:** "Crie um `Makefile` que gere o analisador léxico, o sintático, compile todos os arquivos C/C++ e gere um executável chamado `g-v1` usando o comando padrão `make`[cite: 214, 215, 216, 217]."
